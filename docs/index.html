Authentication Flow

1

Registration

User submits username, email, and password ‚Üí Server validates input ‚Üí Password hashed with bcrypt ‚Üí User stored in MongoDB

2

Login

User submits email and password ‚Üí Server finds user in database ‚Üí Password compared using bcrypt ‚Üí userId and email stored in localStorage

3

Dashboard

Frontend reads userId from localStorage ‚Üí Fetches user data from GET /user/:userId ‚Üí Displays personalized information

4

Logout

User clicks logout ‚Üí localStorage cleared ‚Üí Redirected to landing page

API Endpoints

POST

/register

Register a new user account

Request Body: 
{
  "username": "johndoe",
  "email":  "john@example.com",
  "password": "securePassword123"
}
Response: 
{
  "message": "User registered successfully",
  "userId": "507f1f77bcf86cd799439011"
}

/login

Authenticate user credentials

Request Body: 
{
  "email": "john@example.com",
  "password": "securePassword123"
}
Response: 
{
  "message": "Login successful",
  "userId": "507f1f77bcf86cd799439011",
  "email": "john@example.com"
}

This is a learning/demonstration project. For production use, consider implementing:

  • JWT tokens instead of localStorage
  • HTTPS/TLS encryption
  • Rate limiting
  • CSRF protection
  • HTTP-only cookies for session management
  • Input sanitization
